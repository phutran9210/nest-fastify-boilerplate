# Husky Git Hooks Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cài đặt Husky với hai hooks: `pre-commit` (lint-staged Biome + typecheck) và `commit-msg` (commitlint Conventional Commits).

**Architecture:** `husky` quản lý git hooks, `lint-staged` chạy `biome check --write` chỉ trên staged files, `@commitlint/cli` + `@commitlint/config-conventional` validate commit message. `prepare` script tự động cài hooks sau `pnpm install`.

**Tech Stack:** husky, lint-staged, @commitlint/cli, @commitlint/config-conventional, Biome, pnpm.

---

## File Structure

| File | Action |
|------|--------|
| `package.json` | Modify — thêm devDeps, `prepare` script, `lint-staged` config |
| `commitlint.config.js` | Create — CJS config (`module.exports`) |
| `.husky/pre-commit` | Create (auto bởi husky init) — lint-staged + typecheck |
| `.husky/commit-msg` | Create — commitlint |

### Task 1: Cài packages và khởi tạo Husky

**Files:**
- Modify: `package.json`

- [ ] **Step 1: Cài devDependencies**

```bash
pnpm add -D husky lint-staged @commitlint/cli @commitlint/config-conventional
```

Expected: 4 packages thêm vào `devDependencies` trong `package.json`.

- [ ] **Step 2: Khởi tạo husky**

```bash
pnpm exec husky init
```

Lệnh này tạo thư mục `.husky/` và file `.husky/pre-commit` mặc định (chứa `npm test`), đồng thời thêm `"prepare": "husky"` vào `package.json`.

- [ ] **Step 3: Verify `prepare` script được thêm vào package.json**

```bash
cat package.json | python3 -c "import sys,json; p=json.load(sys.stdin); print('prepare:', p.get('scripts',{}).get('prepare','MISSING'))"
```

Expected: `prepare: husky`

- [ ] **Step 4: Verify thư mục `.husky/` được tạo**

```bash
ls .husky/
```

Expected: thấy file `pre-commit` (và có thể `_/`)

- [ ] **Step 5: Commit**

```bash
git add package.json pnpm-lock.yaml .husky/
git commit -m "build: cài husky, lint-staged, commitlint"
```

### Task 2: Thêm `lint-staged` config vào `package.json`

**Files:**
- Modify: `package.json`

- [ ] **Step 1: Thêm `lint-staged` config vào `package.json`**

Mở `package.json`, thêm block `"lint-staged"` ở top-level (ngang hàng với `"scripts"`, `"dependencies"`):

```json
"lint-staged": {
  "*.{ts,js,json}": [
    "biome check --write --no-errors-on-unmatched"
  ]
}
```

Expected: `JSON OK`

- [ ] **Step 3: Commit**


### Task 3: Tạo `commitlint.config.js`

**Files:**
- Create: `commitlint.config.js`

> **Lưu ý:** `package.json` không có `"type": "module"` → CJS. Dùng `module.exports`, không phải `export default`.

- [ ] **Step 1: Tạo file `commitlint.config.js`**

```js
module.exports = { extends: ['@commitlint/config-conventional'] }
```

- [ ] **Step 2: Verify commitlint hoạt động với valid message**

```bash
echo "feat: test message" | pnpm commitlint
```

Expected: không có output lỗi, exit code 0.

- [ ] **Step 3: Verify commitlint reject invalid message**

```bash
echo "invalid commit message" | pnpm commitlint
```

Expected: output lỗi kiểu `subject may not be empty` hoặc `type may not be empty`, exit code non-zero.

- [ ] **Step 4: Commit**

```bash
git add commitlint.config.js
git commit -m "build: thêm commitlint config (Conventional Commits)"
```

### Task 4: Cấu hình `.husky/pre-commit` và `.husky/commit-msg`

**Files:**
- Modify: `.husky/pre-commit`
- Create: `.husky/commit-msg`

- [ ] **Step 1: Ghi đè `.husky/pre-commit`**

```bash
cat > .husky/pre-commit << 'EOF'
pnpm lint-staged
pnpm typecheck
EOF
```

- [ ] **Step 2: Tạo `.husky/commit-msg`**

```bash
cat > .husky/commit-msg << 'EOF'
pnpm commitlint --edit "$1"
EOF
```

- [ ] **Step 3: Verify nội dung cả hai file**

```bash
cat .husky/pre-commit && echo "---" && cat .husky/commit-msg
```

Expected:
```
pnpm lint-staged
pnpm typecheck
---
pnpm commitlint --edit "$1"
```

- [ ] **Step 4: Commit**

```bash
git add .husky/pre-commit .husky/commit-msg
git commit -m "build: cấu hình pre-commit (lint-staged + typecheck) và commit-msg (commitlint)"
```

### Task 5: Verify end-to-end

- [ ] **Step 1: Test pre-commit hook bằng cách stage file và commit**

```bash
# Tạo file test tạm
echo "" >> src/main.ts
git add src/main.ts
git commit -m "test: verify pre-commit hook"
```

Expected: hook chạy `lint-staged` (Biome check src/main.ts) và `typecheck`, commit thành công.

Revert sau khi test:
```bash
git revert HEAD --no-edit
```

- [ ] **Step 2: Test commit-msg hook với invalid message**

```bash
echo "" >> src/main.ts
git add src/main.ts
git commit -m "bad commit message without type"
```

Expected: commit bị block với lỗi commitlint kiểu:
```
✖   subject may not be empty [subject-empty]
✖   type may not be empty [type-empty]
```

Cleanup:
```bash
git restore src/main.ts
git restore --staged src/main.ts
```

- [ ] **Step 3: Test commit-msg hook với valid Conventional Commit**

```bash
echo "" >> src/main.ts
git add src/main.ts
git commit -m "chore: verify commitlint hook"
```

Expected: commit thành công.

Revert:
```bash
git revert HEAD --no-edit
```

- [ ] **Step 4: Final commit nếu có fix**

